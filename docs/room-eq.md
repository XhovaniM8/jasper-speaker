# Room EQ with REW + CamillaDSP

Apply room correction to the Jasper speaker system via the web dashboard.

---

## Overview

The Room EQ section of the webui lets you:
- Import parametric EQ filters generated by REW (Room EQ Wizard)
- Save named profiles and switch between them instantly
- Reset to flat (no correction) at any time

All filters are applied to CamillaDSP in real time via WebSocket and written to the
live config file so they survive a CamillaDSP restart.

---

## 1. Measure your room

**Option A — HouseCurve (phone mic, ~$20/yr)**

1. Download HouseCurve on iOS/Android
2. Play pink noise from the Jasper system (via MA or the webui Test tab)
3. Hold your phone at the listening position
4. HouseCurve measures and generates a target-matched EQ curve
5. Export as "Parametric EQ" text file

**Option B — REW (free, dedicated measurement mic)**

1. Download REW from roomeqwizard.com
2. Connect a calibrated measurement mic (UMIK-1 or similar) to your laptop
3. Use REW's measurement sweep at the listening position
4. In REW: EQ → AutoEQ → generate parametric filters
5. Export: File → Export Filters → "REW Filter Transfer Function" (.txt)

REW format expected by the importer:

```
Filter 1: ON PK Fc 62.5 Hz Gain -3.0 dB Q 2.00
Filter 2: ON PK Fc 125 Hz Gain 2.5 dB Q 1.50
Filter 3: ON LS Fc 80 Hz Gain -2.0 dB Q 0.71
...
```

Supported filter types: `PK` (Peaking), `LS` (Low shelf), `HS` (High shelf),
`LP` (Low-pass), `HP` (High-pass), `NO` (Notch).

---

## 2. Import filters via the webui

1. Open the Jasper dashboard: `http://<pi-ip>:8080`
2. Scroll to **Room EQ** → **Room Correction** card
3. Click **Choose file** and select your REW `.txt` export
4. Click **Import & Apply** — filters are applied immediately to CamillaDSP

The filter table updates to show the imported filters (highlighted in blue).

---

## 3. Save as a profile

After importing, give the profile a name and click **Save Profile**. Profiles are stored
in `audio/profiles/` in the repo directory and persist across reboots.

To switch between profiles, select one from the dropdown and click **Load**.

To remove a profile, select it and click **Delete**.

---

## 4. Reset to flat

Click **Reset to Flat** to remove all `rew_*` filters from the CamillaDSP pipeline
without touching the crossover or other custom filters.

---

## How it works under the hood

The importer:
1. Parses REW text lines into biquad coefficients
2. Removes any existing `rew_*` filters from the CamillaDSP config
3. Adds new `rew_ch0` through `rew_chN` biquad filters to the `filters` section
4. Injects each filter into the names list of every pipeline channel
5. Applies via `SetConfig` on CamillaDSP's WebSocket (port 1234)
6. Writes the updated config to `~/.config/camilladsp/config.yml`

The filter naming convention (`rew_*`) is what lets the importer surgically replace only
the correction filters while leaving crossover and any manually added filters untouched.

---

## Notes

- A single test speaker on a table is not ideal for room correction — the measurement
  reflects reflections from nearby surfaces more than the room itself. This workflow is
  about learning the measurement → filter → DSP pipeline.
- Measure at the primary listening position with all speakers playing (or just the
  full-range driver at this stage).
- Start conservative: small gains (±3 dB), high Q (1.5-2.0) for narrow corrections.
  Wide boosts can cause clipping or driver damage.
- CamillaDSP applies filters in sample-domain; excessive gain stacking can clip
  internally. Keep the sum of boost filters under 6 dB total.
